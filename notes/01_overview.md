# Agentic RL 领域总览

## 什么是 Agentic RL？

Agentic RL 是指使用强化学习训练 LLM，使其能够**自主使用工具**（搜索引擎、代码解释器、API 等），在**多轮交互**中完成复杂任务。

### 与标准 RLHF 的核心区别

| 维度 | 标准 RLHF | Agentic RL |
|------|-----------|------------|
| **交互轮次** | 单轮生成 | 多轮（think → act → observe 循环） |
| **动作空间** | 仅文本生成 | 文本 + 工具调用 + 环境反馈 |
| **奖励信号** | 神经奖励模型（人类偏好） | 规则化/可验证奖励（正确性、格式） |
| **轨迹长度** | 一次回复 | 多次交互循环 |
| **价值模型** | 需要学习的 Critic | 通常不需要（GRPO） |
| **训练目标** | 对齐人类偏好 | 使用工具解决任务 |
| **训练基建** | 标准 RL 训练循环 | 需要并行环境（搜索引擎、沙箱等） |
| **主流框架** | TRL / DeepSpeed-Chat | veRL（最常用）、自定义 Ray 方案 |

## 技术路线收敛

业界已经收敛到一套共同的 Recipe：

1. **起点**：选择一个有能力的 base/instruct 模型（Qwen2.5-7B 是社区最爱）
2. **可选的冷启动 SFT**：在工具使用示范数据上微调
3. **RL 训练**：使用 GRPO（有时 PPO/REINFORCE++），搭配规则化奖励
4. **训练框架**：veRL + vLLM 推理加速
5. **分阶段奖励设计**：先学格式合规，再学任务准确性

## 核心洞察

1. **纯 RL 可以涌现推理能力**（DeepSeek-R1-Zero 证明了这一点）
2. **规则化奖励优于神经奖励模型**（避免 reward hacking）
3. **GRPO 替代 PPO**：不需要 Critic 模型，节省 50% 显存
4. **小模型蒸馏 > 小模型 RL**：DeepSeek 发现小模型做 RL 不如从大模型蒸馏
5. **冷启动很重要**：纯 RL 虽然能工作但有可读性问题，冷启动 SFT + RL 是最佳组合
6. **多轮信用分配是核心挑战**：跨多个工具调用归因奖励非常困难
